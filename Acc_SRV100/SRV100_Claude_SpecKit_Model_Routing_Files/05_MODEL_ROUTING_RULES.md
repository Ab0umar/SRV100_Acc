# Model Routing Rules

## Claude
Use for:
- Starting the project
- Writing SpecKit /specify, /plan, /tasks
- Writing prompts for every task
- Assigning tools/models
- Reviewing outputs
- Final approval

Do not use for:
- Heavy code
- Bulk refactoring
- Large repetitive extraction

## Cursor
Use for:
- Opening the real project
- Applying multi-file edits
- Frontend/backend integration
- Running project tests

## Codex
Use for:
- Implementing tasks
- Refactoring
- Fixing bugs
- Applying code changes

## GPT-5
Use for:
- API design
- SQL logic
- Report logic
- Complex debugging
- Code draft when needed

## GPT-5 mini / GLM / Kimi / OpenRouter cheap models
Use for:
- Summarizing legacy files
- Extracting fields
- Drafting repetitive SQL
- Checking report lists
- Cheap batch tasks

## Gemini
Use for:
- UI layout
- UX review
- Component visual structure
- Design alternatives

## Ollama / Continue
Use for:
- Local small tasks
- Simple summaries
- Offline edits
